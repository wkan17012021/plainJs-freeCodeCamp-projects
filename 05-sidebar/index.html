This one was easy as we build on lessons learned from previous apps.

Originally, I had the same event listener callback function performing the toggle of the show-sidebar classname.

I decided to refactor this by collecting both buttons into a single variable. It returned a HTML collection where some methods wouldn't work like forEach()

I researched how to manipulate HTML collections and the for loop was what I needed.

I used this to apply the event listener to both buttons thereby reducing repetition.

Improvements

Although the hamburger icon and close icon work to toggle the sidebar, I should have used classList.remove() to remove the classname when the close icon is clicked. This is the solution that was presented in the video tutorial.

It's possible that even when the sidebar is off-screen, if somehow the close icon click event was fired, the pattern of opening and closing of the sidebar would be broken.
