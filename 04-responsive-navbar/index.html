Wasn't initially sure how to do this as it seems this could be done with css and media queries? Set classnames for the list items to display nonefor mobile - display blockfor tablet / desktop?

Watching the first minute or so into the video, John eludes to toggling the show-links classname on and off when the breakpoint for mobile is reached using js to trigger the styling.

I went ahead and tried this by searching for the event type "resize" which you can listen on the window obj. Then create a function that fires but first calculates the viewer's browser width with document.documentElement.clientWidth

Compare this value against the sm breakpoint of 800px, if its less than this then toggle the classname with element.classList.toggle("show-links") In fact, this was a misunderstanding of the project requirements. The mobile menu should only be toggled at small breakpoint if the user clicks on the hamburger icon. This can be achieved with classList.toggle("show-links"). Nonetheless, this app gave me the opportunity to explore window.resize

Improvements

I noticed that the resizing browser is laggy because the event is being fired everytime the browser is resized. Even slight changes in the mobile breakpoint causes the mobile nav to re-render creating a janky UX.

Some suggestions to throttle or debounce the event listener can be found here
