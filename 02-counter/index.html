John's solution was more succinct. I added individual functions and event listeners to assign the button features.

It would have been more concise (D-R-Y) to use querSelectorAll for all buttons using a shared class, loop over them to identify which button was clicked and apply the appropriate functionality.

Check out his source code for how he solved this issue:

  1. Using querySelectorAll to grab all elements with btn classname. Note that a nodelist is returned which is array-like, and affords limited manipulation methods like forEach() which he uses to iterate over.
  2. Now we can pass in to the call back function, on each button, attach an event listener of type 'click'.
  3. He then identifies which button was clicked but interrogating the event object by logging e.currentTarget.
  4. With currentTarget, you then have access to the classlist via e.currentTarget.classList
  5. Use an if/else statement to check if the variable saved to e.currentTarget.classList contains a 'decrease', 'reset', 'increase' class, then apply the appropriate functionality
